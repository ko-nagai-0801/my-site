## TL;DR
- **frontmatter** を埋める（特に `slug` と `date`）
- 画像は **16:10（1600×1000）** を基準に作る
- 見出し→手順→結果→メモ の順にすると読みやすい

## このテンプレの使い方（コピペ手順）
1. このファイルをコピペして `content/blog/2026-01-29-<slug>.mdx` を作成  
2. frontmatter の `title / slug / date / description / tags` を変更  
3. `slug` が **ファイル名の `<slug`> と一致**しているか確認  
4. 文章を埋める（不要なセクションは削除OK）  
5. 下書きなら `draft: true` を付ける（公開する時に消す）

## 画像の扱い（推奨）
### ✅ サムネ（OGP/カード用の基準）
- 比率：**16:10**
- 推奨：**1600×1000**
- 形式：`webp` 推奨
- 置き場所例：`public/images/blog/<slug>.webp`

### ✅ 記事内画像（本文）
**基本はMarkdown記法でOK**（あなたのMDX環境側で `img` を next/image 化している場合、そのまま恩恵を受けられます）

```md
![画像の説明](/images/blog/mdx-template.webp)
```

<MdxImage
  src="/images/blog/noimage.webp"
  alt="テスト画像"
  caption="MdxImage テスト"
  ratio="16/10"
  maxWidth="lg"
/>


※ もし `MdxImage` コンポーネントを直接使う運用なら、あなたの `MdxImage` のpropsに合わせてこう書きます（props名は実装に合わせて調整）：

```mdx
{/* 例：propsが src/alt/caption を受け取る想定 */}
{/* <MdxImage src="/images/blog/mdx-template.webp" alt="画像の説明" caption="キャプション" /> */}
```
